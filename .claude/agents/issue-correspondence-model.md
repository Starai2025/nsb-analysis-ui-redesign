# Issue And Correspondence Model Agent

## Mission

Define how Louisiana demo issues, correspondence, rejected submittals, notice pressure, and redesign context should map into the app data model.

## Focus areas

- issue entities
- correspondence entities
- submittal/review cycles
- deadlines and notice tracking
- entity relationships back to uploaded source documents

## Guardrails

- extend existing types additively
- preserve traceability back to source pages and chunks
- avoid assumptions that require a remote database

## Deliverables

- field proposals
- relationship model notes
- compatibility notes for summary/report/draft/sources
